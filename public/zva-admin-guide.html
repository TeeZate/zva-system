System Overview
What the ZVA Digital Management System is and how it is structured

The ZVA Digital Management System is a centralised web platform that manages all aspects of Zimbabwe Volleyball Association's operations — from league fixtures and live scores to player registration, news publishing, and match statistics.

The system runs entirely in the cloud. There is nothing to install. Any administrator with an internet connection and a modern browser can access it from a laptop, tablet, or smartphone.

System URL

The system is accessible at:

https://zva-system.up.railway.app

The public website and the administrator panel share the same address. Administrators access their panel by navigating to /admin.

Two Parts of the System

🌐
Public Website
Visible to everyone — fans, media, players, and sponsors. Shows fixtures, live scores, standings, player profiles, news, and tournament results. No login required.
🔐
Administrator Panel
Accessible only to authorised ZVA staff. Used to enter scores, manage teams, publish news, handle O2 registrations, and monitor the system. Login required.

How Data Flows

Admin enters data
→
Saved to database
→
Public site updates instantly
→
Fans see it live

All data is stored in a secure cloud database (Supabase). When an admin updates a score or publishes an article, the change appears on the public website within seconds — no manual refresh or file upload is required.

Key Features at a Glance

  • Live score entry with real-time scoreboard display — usable on a TV or projector courtside
  • COURT scouting app for statisticians — records every action per player, synced live
  • Full fixture management for Premier League, ZVA Cup, Junior Championship, and more
  • Automatic standings calculation and league tables
  • Player database with national team flags and career statistics
  • News publishing with categories, featured articles, and SEO-ready pages
  • O2 form upload portal for team administrators
  • System health monitor so technical issues are spotted immediately
ℹ️
Built for reliability
The system is hosted on Railway (cloud server) with automatic deployments. The database is hosted on Supabase with daily backups. If you ever see a technical error, check the System Monitor page first — it will tell you exactly which service is affected.

Chapter Two
Roles & Permissions
There are four access levels — each person only sees what they need

The ZVA system uses a role-based access model. When an administrator logs in, the system automatically detects their role and shows only the pages they are permitted to use. There is no way for a team admin to accidentally access super admin functions.

Role Who holds it What they can do
Super Admin ZVA technical officer / secretary general Everything — full system access including creating/disabling other admin accounts, system monitor, all content management
Admin ZVA competition managers Manage matches, live scoring, teams, players, tournaments, news. Cannot manage other admin accounts or view system health
Team Admin Club registrars / team managers Access their own team dashboard only. Can upload O2 registration forms for their team. Cannot see any other team's data
Statistician Match statisticians / scorers Access the COURT scout app only. Can select a match and record live statistics. Cannot edit fixtures or publish news
⚠️
Only Super Admins can create other admin accounts
If a team admin or statistician needs their password reset or account disabled, only a Super Admin can do this. Keep the Super Admin credentials in a secure place.

What Happens When Each Role Logs In

SA
Super Admin / Admin — Full Admin Panel
Sees the complete sidebar: Dashboard, Matches, Live Scoring, COURT Scout, Teams, Players, Tournaments, News, Admin Users, and System Monitor.
TA
Team Admin — Team Portal Only
After login they are automatically redirected to /admin/team. They see their team dashboard and O2 forms page. There is no way to navigate to other teams or admin functions.
ST
Statistician — Scout App Only
After login they are automatically redirected to /admin/scout. They see a list of upcoming matches and can open the COURT app to record live stats. No other pages are accessible.

Logging In & Navigation
How to access the admin panel and find your way around

Accessing the Admin Panel

1
Open your browser
Go to https://zva-system.up.railway.app/admin/login. Use Chrome, Edge, Firefox, or Safari. The system works on desktop and mobile.
2
Enter your credentials
Type your email address and password. These are issued to you by the Super Admin. If you have not received credentials, contact the ZVA technical officer.
3
You are redirected automatically
Based on your role, you will land on the correct page immediately. Super Admins and Admins go to the Dashboard. Team Admins go to their Team Portal. Statisticians go to the COURT Scout app.
💡
Bookmark the admin panel
Save https://zva-system.up.railway.app/admin as a bookmark on your device. The system will remember your login session for several days so you will not need to sign in each time.

Admin Panel Sidebar Navigation

The left sidebar is your main navigation. Here is what each section does:

Sidebar ItemPurpose

MatchesView, create, and edit all fixtures
Live ScoringEnter scores point-by-point for live matches

Signing Out

Click Sign Out at the bottom of the sidebar. Always sign out when using a shared or public computer. Your session expires automatically after a period of inactivity.

Managing Matches & Fixtures
Scheduling matches, recording results, and managing tournaments

Viewing All Fixtures

Go to Matches in the sidebar. You will see a list of all fixtures — past, upcoming, and live — with filters by tournament, status, and date. Each match card shows both teams, the scheduled time, venue, and current status.

Scheduling a New Match

1
Click "New Match"
The button is at the top right of the Matches page.
2
Fill in the match details
Select the tournament, home team, away team, venue, date, and time. All teams and venues must already exist in the system before you can assign them.
3
Set status to "Scheduled"
The match will appear on the public fixtures page immediately after saving.

Match Status Values

StatusMeaningPublic display
ScheduledMatch is in the future, not yet startedShown in fixtures with date/time
LiveMatch is currently in progressShown with live score and pulsing indicator
FinishedMatch has been completedShows final score and set breakdown
Postponed / CancelledMatch will not take place as scheduledShown with status label, no score

Entering a Final Result Manually

If live scoring was not used during the match, you can enter the result after the fact. Open the match from the Matches page, edit the set scores directly, and set the status to Finished. The standings and league table will update automatically.

Managing Tournaments

Go to Tournaments in the sidebar to create and manage competitions. Each tournament has a name, division (Premier League, ZVA Cup, etc.), season dates, and status. Matches are linked to a tournament — you must create the tournament first before scheduling matches inside it.

ℹ️
Standings update automatically
When you mark a match as Finished with a final score, the standings table for that tournament recalculates automatically — wins, losses, sets ratio, and league points are all updated. You do not need to update standings separately.

Chapter Five
Live Score Entry
Update scores in real time — the public scoreboard refreshes instantly for all viewers

The ZVA system includes a real-time scoring interface. When an admin enters a score update, it is broadcast immediately to every person viewing the public scoreboard — including on TV screens and projectors at the venue. No page refresh is needed on any device.

Starting a Live Match

1
Go to Live Scoring in the sidebar
You will see two sections: "Currently Live" and "Scheduled — Ready to Start".
2
Click "Start Match" next to the fixture
This opens the score entry screen and changes the match status to Live on the public website.
3
Use the score entry controls
Tap the +1 button for the team that won the point. The score updates on the scoreboard in under one second for everyone watching.
4
End each set
When a set finishes, click End Set. The scores reset to zero and the set counter advances. Set scores are saved and shown in the history.
5
Match concludes automatically
When one team reaches 3 set wins (best of 5) the system marks the match as finished. The final result is saved and the public scoreboard shows the completed score.

The Public TV Scoreboard

Every live match has a dedicated full-screen scoreboard page at:

https://zva-system.up.railway.app/scoreboard/[match-id]

Open this URL on any screen connected to a TV or projector at the venue. It displays team names, current set scores, the score in large text, and a live indicator. It updates automatically — no one needs to touch the TV screen.

💡
Using both Live Scoring and COURT Scout together
If a statistician is using the COURT Scout app for the same match, they do not need to be coordinated with the Live Scoring admin. COURT Scout sends score updates automatically every time a point is awarded — so the scoreboard stays correct from both sources. However, you should use one method at a time to avoid duplicate updates.

Correcting a Score Mistake

If a score is entered incorrectly, go to the match in the Matches page, click Edit, and manually adjust the scores. The scoreboard will update within seconds. There is no need to end and restart the match.

Chapter Six
COURT Scout — Live Stat Collection
How statisticians record per-player actions and sync data to the live scoreboard

COURT is the ZVA's integrated scouting and statistics application. It runs directly inside the ZVA admin panel and is designed to be used courtside on a tablet or laptop. Statisticians record every action — attacks, serves, blocks, digs — for each player. The scores sync to the live scoreboard automatically.

Who Uses COURT Scout

Users with the Statistician role are taken directly to COURT Scout when they log in. Admins and Super Admins can also access it via the sidebar.

Starting a Scouting Session

1
Open COURT Scout in the admin panel
Click COURT Scout in the sidebar. You will see a list of all scheduled and live matches.
2
Select the match you are recording
Click the match card. The COURT app opens in an embedded panel below, already connected to that match. The ZVA badge at the top of COURT will show "ZVA READY" in green.
3
Click "LOAD ROSTER"
This imports the team names and player lists from the ZVA database into the COURT app automatically. You do not need to type in any names.
4
Click "Start Match" in COURT
The RECORD tab opens. You are now ready to record stats.

Recording Statistics

On the RECORD screen:

  1. Select a player from the chip list at the top (home team left, away team right)
  2. Select the action — Attack, Serve, Block, Dig, Receive, Set, etc.
  3. For attacks and serves, select the type (e.g. Jump Serve, Pipe, etc.)
  4. Select the outcome — Good ✓, OK ~, or Error ✗
  5. Click LOG ACTION — the event is saved and synced

To award a point manually, use the +1 HOME / +1 AWAY buttons at the top of the screen. This is the primary way scores advance.

Ending a Set and Concluding the Match

ActionHowWhat happens
End a set Click END SET Confirms set result via a modal. Scores reset. ZVA scoreboard updates. Set history is saved.
Match won naturally Automatic when a team reaches 3 set wins A 🏆 Match Complete overlay appears showing the final result. The match is marked Finished in the ZVA database.
End match early Click the red END MATCH button Use for forfeits or retirements. Confirms via modal then completes the match with the current set tally.
ℹ️
COURT works offline too
COURT stores all events on the device locally (using the browser's built-in storage). If the internet drops mid-match, stat recording continues uninterrupted. However, live score updates to the ZVA scoreboard will pause until the connection is restored — these are sent automatically once the connection returns.

Viewing and Exporting Statistics

The LOG tab in COURT shows a full event-by-event breakdown, stat summary cards, and a per-player performance table. To export data for FIVB reporting, click the ↓ CSV button — this downloads a spreadsheet of all recorded actions.

Full-Screen Mode

For courtside use on a dedicated tablet, click Full Screen at the top of the Scout page. This hides the admin panel and shows only COURT — ideal for a statistician who only needs the scouting interface.

Teams & Players
Managing club profiles, rosters, and the national team

Managing Teams

Go to Teams in the sidebar to view and manage all registered clubs. Each team record contains:

  • Name and short name (e.g. "Harare City Volleyball Club" / "HARARE")
  • Division — Premier League, Division One, Women's, Junior, etc.
  • Province and city
  • Home venue — linked to the venues database
  • Coach name, colours, and founding year
  • Logo — upload via the team edit page
⚠️
Add teams before scheduling matches
You cannot create a fixture without both teams already existing in the system. Always add new clubs to the Teams database before the season starts.

Managing Players

Go to Players in the sidebar. Each player profile contains:

  • First and last name, jersey number, position
  • Date of birth, height, nationality
  • Team assignment
  • National team flag (tick this for players who represent Zimbabwe nationally)
  • Career statistics — points, aces, blocks (used for Hall of Fame displays)
  • Player photo (uploaded URL)

Player Positions

Full NameShort Code
SetterS
Outside HitterOH
Middle BlockerMB
OppositeOPP
LiberoLIB
Defensive SpecialistDS

Team Rosters in COURT Scout

When a statistician selects a match in COURT Scout and clicks Load Roster, the system automatically pulls the players from both teams' rosters in the database. This means keeping the Players database up to date is essential for accurate stat collection. If a player is not in the system, they cannot be tracked individually in COURT.

News & Announcements
Publishing articles, match reports, transfer news, and official announcements

The ZVA system includes a full news publishing platform. Articles appear on the public website under the News section and are visible to everyone. There is no character limit on article content.

Publishing a New Article

1
Go to News → New Article
Click News in the sidebar then New Article at the top right.
2
Fill in the article details
  • Title — the headline shown on the website
  • Slug — the URL identifier (auto-generated from the title, e.g. zva-season-preview-2026)
  • Category — Match Report, Announcement, Transfer, National Team, or General
  • Excerpt — a short summary shown on the news listing page (2–3 sentences)
  • Content — the full article body
  • Thumbnail URL — link to a cover image
  • Tags — keywords for filtering (e.g. "premier-league", "harare")
  • Featured — tick this to pin the article at the top of the news page
3
Click Publish
The article appears on the public website immediately at /news/[slug].

Article Categories

CategoryUse for
Match ReportPost-match write-ups covering key moments and standout players
AnnouncementOfficial ZVA statements, rule changes, season calendars
TransferPlayer signings, club moves, loan deals
National TeamZimbabwe national team news, FIVB events, training camps
GeneralAll other volleyball news
💡
Only one featured article at a time
Mark one article as Featured to pin it at the top of the news page with a large hero banner. If you feature a new article, unfeature the previous one to keep the layout clean.

Chapter Nine
Team Portal — O2 Form Uploads
How clubs submit FIVB player registration documents digitally

The O2 form is the official FIVB document used to register players for international and national competition. The ZVA system allows team administrators to upload their O2 forms digitally, replacing the need to deliver paper copies to the ZVA office.

What Team Admins See

When a Team Admin logs in, they go directly to their Team Portal. They see:

  • Their club's profile card (name, division, coach)
  • A summary of their O2 submissions (pending, approved, rejected)
  • A list of all previous uploads with status badges
  • A link to upload a new O2 form

Uploading an O2 Form (Team Admin)

1
Go to O2 Forms in the Team Portal sidebar
Click O2 Forms in the left nav.
2
Drag and drop or select the file
Accepted formats: PDF, JPG, PNG, WEBP. Maximum file size: 10 MB. Scan quality must be sufficient for ZVA staff to read all fields clearly.
3
Select the season and enter the player count
Choose the relevant season (e.g. 2025/2026) and enter how many players are listed on this O2 form.
4
Click Submit
The form is uploaded and its status is set to Pending. A ZVA admin will review it and update the status to Approved or Rejected.

O2 Submission Statuses

StatusMeaning
PendingUploaded, awaiting ZVA review
ApprovedZVA has accepted the registration
RejectedZVA has rejected the form — reviewer notes will explain why. Re-upload a corrected version.
⚠️
Reviewing O2 forms (Super Admin / Admin)
ZVA staff review O2 submissions from within the admin panel. Go to Teams → [Team Name] → O2 Uploads to see all pending submissions. Open the file, review it, then set the status to Approved or Rejected and add notes if needed.

Chapter Ten
Managing Admin Accounts
Creating, disabling, and resetting passwords — Super Admin only

All admin account management is done from Admin Users in the sidebar. Only Super Admins can access this page.

Creating a New Admin Account

1
Click "Add Admin"
A modal dialog opens.
2
Enter their email address
Use the person's official email. An invitation will be sent to this address.
3
Select their role
Choose from Team Admin, Statistician (COURT Scout), or Super Admin. For Team Admins, also select which club they manage.
4
Click Send Invite
The person receives an email with a link to set their own password. Once they complete registration, their account is active.

Resetting a Password

On the Admin Users page, find the person's row and click the key icon. This sends a password reset email to their registered address. They click the link in the email and set a new password.

Disabling an Account

Click the shield icon next to the user's name to disable their account. A disabled account cannot log in — the person is locked out immediately. This is the correct action when someone leaves the organisation. You can re-enable the account at any time by clicking the same button.

🔒
Disable accounts promptly when staff change
When a team manager changes, a statistician leaves, or any admin departs, disable their account immediately. Do not wait. An active account belonging to a former staff member is a security risk.

System Monitor & Health Dashboard
Checking the health of all services — Super Admin only

The System Monitor (accessible via the sidebar under the System section) gives a live view of all backend services. It is the first place to check whenever something seems slow or broken.

Health Checks Displayed

ServiceWhat it checks
DatabaseCan the system read from Supabase? Measured in milliseconds.
AuthenticationIs the login service responding correctly?
StorageIs the file storage system (for O2 forms) available?
RealtimeIs the live score broadcast service working?
EnvironmentAre all required configuration keys present on the server?

Status Indicators

IndicatorMeaningAction
OKService is healthy and responding normallyNone required
WARNService is responding but slower than expectedMonitor — may resolve itself. Contact support if it persists.
ERRORService is not responding or has failedContact your technical support contact immediately

System Event Log

Below the health checks, a log shows the last 50 system events — including failed login attempts, upload errors, and API failures. This helps diagnose issues like a team admin repeatedly failing to log in (possible password issue) or frequent upload failures (possible connectivity problem at their end).

💡
Refresh button
The System Monitor page has a Refresh button in the top right. Click it to re-run all health checks. The page does not auto-refresh — you must click Refresh each time you want updated readings.

Troubleshooting & FAQs
Common issues and how to resolve them quickly
ProblemLikely causeSolution
Cannot log in — "Invalid credentials" Wrong password or email Double-check the email. If forgotten, ask a Super Admin to send a password reset email.
Cannot log in — "Account disabled" Super Admin has disabled the account Contact your Super Admin to re-enable the account.
Score update not appearing on the scoreboard Realtime connection lost Refresh the scoreboard page. Check the System Monitor for Realtime status. Ensure the device running the scoreboard has a stable connection.
COURT Scout shows "ZVA ERROR" Match ID not found in database Ensure the match exists and is in Scheduled or Live status in the admin Matches page.
O2 upload fails File too large or wrong format Ensure the file is PDF, JPG, PNG, or WEBP and under 10 MB. Compress large PDFs before uploading.
Team admin cannot see their team's data Wrong team assigned in admin account A Super Admin must edit the team admin's account and assign the correct team.
Player not appearing in COURT Scout roster Player not linked to that team in the database Go to Players, find the player, and ensure their team assignment is correct.
News article not showing on public site Article may not be published Open the article in the admin News page and confirm it is published (not in draft).
System Monitor shows DB ERROR Supabase outage or configuration issue Check status.supabase.com for any service incidents. If all clear, contact your technical support contact.
Page loads very slowly Poor network or server cold-start Wait 20 seconds and refresh. Railway may be waking up the server from sleep. If persistent, check System Monitor.
📞
Who to contact for technical support
For issues that cannot be resolved using this guide, contact the ZVA system administrator. When reporting a problem, include: the page you were on, what you clicked, and what error message appeared (take a screenshot if possible). This helps diagnose problems faster.
